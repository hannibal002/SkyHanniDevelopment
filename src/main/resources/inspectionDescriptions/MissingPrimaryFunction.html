Reports concrete SkyHanniEvent subclasses that have no @PrimaryFunction annotation and are not abstract.

Without @PrimaryFunction, handlers must supply an explicit eventType = SomeEvent::class argument on @HandleEvent. Annotating the event class enables conventionally-named, parameterless handler functions. 


// Missing annotation - will be flagged:
class SlotClickEvent : SkyHanniEvent()

// Correct:
@PrimaryFunction("onSlotClick")
class SlotClickEvent : SkyHanniEvent()

A quick fix is offered when a candidate name can be derived from the class name and is not already claimed by another event. Suppressable via @Suppress("MissingPrimaryFunction").
