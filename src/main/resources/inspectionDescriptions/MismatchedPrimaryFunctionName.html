Reports @HandleEvent functions whose name does not match the @PrimaryFunction declared on the specified event class.

When @HandleEvent specifies an explicit eventType, the handler function name should match the event's @PrimaryFunction value for consistency with the rest of the codebase. 


// SlotClickEvent declares @PrimaryFunction("onSlotClick")

// Mismatched name - will be flagged:
@HandleEvent(eventType = SlotClickEvent::class)
fun onWrongName() { }

// Correct:
@HandleEvent(eventType = SlotClickEvent::class)
fun onSlotClick() { }

Suppressable via @Suppress("MismatchedPrimaryFunctionName").
