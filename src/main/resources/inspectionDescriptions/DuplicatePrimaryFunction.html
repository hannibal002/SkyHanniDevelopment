Reports SkyHanniEvent subclasses whose @PrimaryFunction name is already declared on another event class.

The name-to-event mapping is 1:1. Duplicate names cause ambiguity in handler dispatch and must be resolved by renaming one of the conflicting @PrimaryFunction values. 


// Duplicate name across two events - one will be flagged:
@PrimaryFunction("onTick")
class ServerTickEvent : SkyHanniEvent()

@PrimaryFunction("onTick")
class ClientTickEvent : SkyHanniEvent()

Suppressable via @Suppress("DuplicatePrimaryFunction").
